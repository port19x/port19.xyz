2. Day 1

2.1. Problem

On day one we have to make sure none of the elves starve by first identifying the top of elf and then the top 3, sorted by calories they have. Input was a long string with numbers on lines. Each line represents one food item. Each empty line represents that the next will be food of another elf.

I spent the majority of my time parsing the input into a neat data structure. Had to look up a lot of stuff in the clojure docs because I don’t know the names of all important functions by heart yet. Getting the second star after my first took less than a minute.

2.2. First star

(->> (slurp "input.txt")
     (#(str/split % #"\n"))
     (partition-by #(= "" %))
     (filter #(not (= [""] %)))
     (map #(map read-string %))
     (map #(apply + %))
     (apply max))

3. Day 2

3.1. Problem

On day two we participate in a rock paper scissors tournament with the elves for a tent spot near the snack storage. An elf gives us a strategy guide. A is Rock, B is Paper, C is Scissors. Since that elf gets interrupted we believe X must be Rock, Y must be Paper and Z must be Scissors. Using this thesis we calculate our score for the first star of the day. The elf returns and corrects us: X means we should loose, Y means we should tie, Z means we should win. With this information we recalculate our score for the second star of the day.

I spent the majority of the time today filling out the cases for the rps function. Since I speculated the star 2 to be about my opponents score, I calculated that too. A satisfying little puzzle today.

3.2. First star

(defn rps
  [input]
  "A,X -> Rock = 1 | B,Y -> Paper = 2 | C,Z -> Scissors = 3 | 0 loose | 3 draw | 6 win"
  (cond (= input "A X") [4 4]
        (= input "A Y") [1 8]
        (= input "A Z") [7 3]
        (= input "B X") [8 1]
        (= input "B Y") [5 5]
        (= input "B Z") [2 9]
        (= input "C X") [3 7]
        (= input "C Y") [8 2]
        (= input "C Z") [6 6]))

(->> (slurp "input.txt")
     (#(str/split % #"\n"))
     (map rps)
     (map last)
     (apply +))

4. Day 3

4.1. Problem

The elves didn’t sort their stuff properly into the comparments of their bags. Afterwards they noticed that their group badge was in there too somewhere. For some reason they needed prioritization too.

Today I had to split strings and intersect character sets. Easy clojure stuff otherwise.

4.2. First star

(defn prioritize
  "a -> 1, z -> 26, A -> 27, Z -> 52"
  [x]
  (cond (Character/isUpperCase x) (- (int x) 38)
        (Character/isLowerCase x) (- (int x) 96)))

(->> (slurp "input.txt")
     (#(str/split % #"\n"))
     (map #(split-at (/ (count %) 2) %))
     (map #(map set %))
     (map #(apply set/intersection %))
     (map vec)
     (flatten)
     (map prioritize)
     (apply +))

4.3. Second star

5. Day 4

5.1. Problem

Today the elves had to clean up but got assigned overlapping section IDs. First we had to check for containment and then for any overlap at all.

Real bracket-salad today.

5.2. First star

(->> (slurp "input.txt")
     (str/split-lines)
     (map #(map read-string (str/split % #"[,-]")))
     (map #(or
            (and (>= (nth % 0) (nth % 2)) (<= (nth % 1) (nth % 3)))
            (and (<= (nth % 0) (nth % 2)) (>= (nth % 1) (nth % 3)))))
     (filter #(identity %))
     (count))

5.3. Second star
